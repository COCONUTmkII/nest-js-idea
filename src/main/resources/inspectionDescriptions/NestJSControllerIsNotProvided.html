Detects NestJS controllers that are not registered in any NestJS module.

This inspection identifies classes decorated with @Controller that are not included in the controllers metadata of any NestJS module.

The inspection supports both:

  • Static module metadata via @Module({...})
  • Dynamic modules returning DynamicModule

Dynamic module support includes patterns such as: 


@Module({})
export class SessionModule {

  static forRoot(): DynamicModule {
    return {
      module: SessionModule,
      controllers: [SessionController],
    };
  }
}

Controllers that are not registered in any module will not be instantiated by NestJS and may lead to runtime errors.

Example: 


import { Controller } from '@nestjs/common';

@Controller()
export class MyController {
  // ...
}

If MyController is not added to any module's controllers metadata, it will be reported by this inspection.

To fix this, register the controller in a module: 


import { Module } from '@nestjs/common';
import { MyController } from './my.controller';

@Module({
  controllers: [MyController],
})
export class MyModule {}

Or inside a dynamic module: 


@Module({})
export class MyModule {

  static forRoot(): DynamicModule {
    return {
      module: MyModule,
      controllers: [MyController],
    };
  }
}
