Detects NestJS modules that are not used anywhere in the application.

This inspection identifies classes decorated with @Module that are not referenced in any other module's imports array and are not used as application root modules via NestFactory.create().

A module is considered used if:

  • It is imported by another module (imports: [...])
  • It is used as a root module in application bootstrap (NestFactory.create(AppModule))

Both static and dynamic NestJS modules are supported: 


@Module({
  imports: [UserModule],
})
export class AppModule {}


@Module({})
export class SessionModule {

  static forRoot(): DynamicModule {
    return {
      module: SessionModule,
      imports: [DatabaseModule],
    };
  }
}

The inspection also supports dynamic bootstrap detection: 


await NestFactory.create(AppModule);

Modules that are not referenced anywhere in the module graph and are not used as application entry points are considered unused and will be reported.

Example of an unused module: 


import { Module } from '@nestjs/common';

@Module({})
export class UnusedModule {}

To fix this, ensure the module is either:

  • Imported in another module's imports array
  • Used as a root module in NestFactory.create()
